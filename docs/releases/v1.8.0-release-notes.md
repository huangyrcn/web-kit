# v1.8.0 release notes

## Summary

Adds a unified **Caddy authentication gateway** in front of the web-kit
backend's three outward services (SearxNG, Chrome CDP, noVNC), replacing the
previous LAN-wide unauthenticated exposure with a single shared
`WEB_KIT_API_KEY`. Scripts authenticate via the `X-API-Key` header; noVNC uses
HTTP Basic Auth (username `webkit`, password = the same key).

## What changed

### Backend (gateway)
- New `backend/Caddyfile` — three listeners (8082/9223/6080), each gated on the
  shared key; SearxNG/CDP via `X-API-Key` header, noVNC via bcrypt Basic Auth.
- New `backend/start-caddy.sh` — fail-closed launcher; refuses to start if
  `WEB_KIT_API_KEY` or `WEB_KIT_NOVNC_HASH` is empty.
- `Dockerfile` installs Caddy 2.8; `supervisord.conf` runs it first
  (priority 5). Internal services moved off the outward ports: cdp-forward
  9223→9224, novnc 6080→6081 (SearxNG stays 8080).
- `docker-compose.yml` requires `WEB_KIT_API_KEY` / `WEB_KIT_NOVNC_HASH`
  (`${VAR:?}`), publishes only the gateway ports.

### Plugin (clients)
- `ask-search`, `cdp-download`, `crwlr` now read `WEB_KIT_API_KEY`, fail-closed
  when missing, and attach it (`X-API-Key` header; CDP HTTP + WS both carry it;
  crwlr injects via a `connect_over_cdp` monkeypatch).
- `web_kit_api_key` added to `plugin.json` userConfig alongside the two URLs.

### Fail-closed at three layers
Launcher (empty key → refuse), compose (`${VAR:?}` → error), and each script
(missing key → exit before any request).

Prior config backed up on the NAS (`.bak-<timestamp>/`) for rollback.

## Upgrade notes (breaking)

This is a **fail-closed cutover**: once the gateway is live, every client must
set `WEB_KIT_API_KEY` or it gets 401.

1. On the backend host: generate `WEB_KIT_API_KEY` (`openssl rand -hex 32`) and
   `WEB_KIT_NOVNC_HASH` (`caddy:2.8 caddy hash-password`), add both to `.env`
   (escape `$`→`$$` in the hash for compose), then `docker compose build && up -d`.
2. On each agent host: set `web_kit_api_key` in the plugin userConfig (or export
   `WEB_KIT_API_KEY`) to the same value.
