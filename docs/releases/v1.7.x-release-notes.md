description so `ask-search` is more likely to trigger for genuine web-search
tasks and less likely to misfire for local-file / code / download tasks.

## Why

The v1.6.1 body describes the skill as a full chain:
trigger → input → tools → workflow → quality → errors → output.

That part was already good.

But the `description` field is the primary trigger signal for deciding whether
Claude should consult the skill at all. v1.7.0 had a trigger-oriented
description already, but the optimization loop can further tune the wording
to improve recall without blowing up false positives.
